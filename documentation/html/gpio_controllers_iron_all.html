GPIO Controllers Iron Port#

Context and Problem#

The Telesoud API requires the gpio_controllers package to function properly. This package provides essential GPIO control capabilities for hardware interface communication.

Issue: The gpio_controllers package was not available for ROS2 Iron distribution, only for ROS2 Humble. Since our system runs on Iron, we needed a compatible version.

Solution: Custom Port#

To resolve this dependency issue, we performed a manual port of gpio_controllers from ROS2 Humble to ROS2 Iron. This custom port is included in this repository under the gpio_controllers_iron_all directory.

Directory Structure#

The gpio_controllers_iron_all directory contains three essential components:

gpio_controllers_iron#

  • Purpose: Main GPIO control package ported to Iron

  • Origin: Adapted from ROS2 Humble gpio_controllers

  • Function: Provides GPIO hardware interface for robot control

  • Modifications: Updated dependencies and build system for Iron compatibility

custom_control_msgs#

  • Purpose: Custom control message definitions

  • Why needed: Dependency of gpio_controllers that required modifications

  • Changes: Message format adaptations for Iron compatibility

  • Content: Custom message types for GPIO control communication

custom_realtime_tools#

  • Purpose: Real-time utilities for control systems

  • Why needed: Another gpio_controllers dependency requiring updates

  • Changes: Real-time processing adaptations for Iron

  • Function: Provides real-time safe utilities for GPIO operations
